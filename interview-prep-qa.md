# 인터뷰 예상 질문 & 답변 예시

## 협업 역량

### Q1. 대규모 프로젝트에서 어떻게 협업하셨나요?

**답변:**
아톤에서 KB국민은행, IBK기업은행 등 제1금융권 대형 프로젝트를 다수 진행하면서 체계적인 협업 경험을 쌓았습니다.

현재 코나아이에서는 MSA 구조의 결제플랫폼을 운영하며, 회원 정보 관리 도메인을 주도적으로 담당하면서도 다른 도메인 팀과 긴밀히 협업하고 있습니다.

특히 가상 순번 대기 시스템 개발 시에는 요구사항 정의부터 아키텍처 설계, 개발, 성능/장애 테스트, 운영 서비스 릴리즈까지 전 과정을 프로젝트 오너로서 주도하면서, 프론트엔드 팀, 인프라 팀과의 협업을 통해 월 1일 대규모 이벤트 기간에 코어 시스템 지연 장애 0건을 달성했습니다.

---

### Q2. 코드 리뷰는 어떻게 진행하시나요?

**답변:**
유지보수성과 코드 품질 향상을 위해 코드 리뷰를 적극 활용합니다.

헥사고날 아키텍처, 클린 아키텍처, DDD 등 소프트웨어 설계 원칙을 실제 프로젝트에 적용하고 있으며, 리뷰 시에는 단순히 버그를 찾는 것뿐만 아니라 아키텍처 적합성, 확장 가능성, 성능 등을 함께 고려합니다.

또한 동료의 고민과 의견에 대해 적극적으로 듣고 함께 고민하면서, 다른 시각으로 바라보고 놓칠 수 있는 이슈는 없을지 함께 분석하는 적극적인 상호 피드백 문화를 만들고자 노력하고 있습니다.

---

### Q3. 팀원들과 지식 공유는 어떻게 하시나요?

**답변:**
새로운 기술을 혼자 공부하고 업무에 적용하는 것이 아니라, 학습한 내용을 세미나를 통해 공유하거나 팀원들과 함께 스터디 모임을 진행하며 함께 성장하는 문화를 만들고자 노력합니다.

대표적인 예로, Kotlin 언어를 도입하기 전에 제대로 이해하고 사용하고 싶다는 생각으로 팀원들과 함께 'Kotlin in Action' 도서를 교재 삼아 스터디 모임을 진행했습니다.

그 결과 단순히 문법을 외우는 것이 아니라 코틀린 언어의 사상과 지향하는 코드 스타일을 이해할 수 있었고, 팀 전체의 코드 품질 향상에도 기여할 수 있었습니다.

### Q4. 프로젝트 진행 중 의견 충돌이 있었던 경험이 있나요?

**답변:**
적극적인 소통을 통해 다양한 피드백을 공유하려 노력합니다.

기술적 의사결정 시에는 현재 시스템의 확장성, 유지보수성, 성능, 비즈니스 요구사항 등을 종합적으로 고려하여 판단합니다.

예를 들어, 가상 순번 대기 시스템 개발 시 외부 솔루션 도입과 자체 개발 사이에서 선택을 해야 했는데, 비용, 커스터마이징 가능성, 운영 난이도 등을 종합적으로 분석하여 팀과 함께 논의한 결과 Redis 기반 자체 개발을 선택했고, 성공적으로 100K 동시접속을 안정적으로 제어할 수 있었습니다.

## 기술 역량

### Q5. 대용량 트래픽을 어떻게 처리하셨나요?

**답변:**
누적 회원 1700만, MAU 400만 이상의 대규모 서비스를 운영하면서 다양한 대용량 트래픽 처리 경험을 쌓았습니다.

**가상 순번 대기 시스템 사례:**
- Redis Sorted-Set 기반 대기열·순번 관리로 동시접속 100K 이상 안정 제어
- Lua 스크립트로 단일 키 트랜잭션·락 최소화 설계
- Sliding-Window로 트래픽 대기/진입 처리 제어
- HTTP Long-Term Polling과 서버 계산 기반 Polling 주기 동적 조절
- 단일 인스턴스 기준 TPS 600, p95 응답시간 3ms 달성
- AWS 클라우드 인프라 활용한 Auto-Scaling 환경 구성

**민생회복 소비쿠폰 시스템:**
- 최대 TPS 1400 대응 가능한 시스템 구축
- 개인정보 암호화 처리 및 불필요한 DB Lock 제거로 성능 최적화

### Q6. 성능 개선 경험을 구체적으로 설명해주세요.

**답변:**
크게 두 가지 대표적인 성능 개선 사례가 있습니다.

**1. 가맹점 연동 Batch 성능 최적화:**
- 처리 시간: 4300s → 24s (약 179배 개선)
- 병목 구간 식별: 파일 IO/DB/외부 API 호출 분석
- Spring Batch Partitioner로 파티셔닝·Step 병렬 처리
- 외부 API 연동을 WebClient 전환하여 Async/Non-Blocking 처리
- Chunk Listener로 후속 비동기 호출 구현
- 재시도 정책 정교화로 운영 안정성 확보

**2. 주소 검색 시스템 최적화:**
- 도로명 검색 성공율: 89.2% → 99.8%
- 지번/관련-지번 검색 성공율: 73.3% → 81.3%
- 평균 응답시간: 18ms → 6ms (3배 개선)
- Filebeat → Logstash → Elasticsearch 데이터 파이프라인 구성
- 도로명/지번/동의어 기반 맞춤 애널라이저 설계
- 유사도 매칭 로직 적용 및 좌표 정합성 검증

### Q7. MSA 환경에서 개발한 경험이 있나요?

**답변:**
코나아이의 결제플랫폼은 MSA 구조로 구성되어 있으며, 다양한 금융 관련 도메인으로 이루어져 있습니다.

**주요 개발 경험:**
- **OpenAPI Gateway 시스템:** Spring Cloud Gateway, Spring Cloud Circuit Breaker 활용하여 API 인증/인가, 데이터 암복호화, 장애 전파 방지 처리
- **Reactive 프로그래밍:** Spring WebFlux 활용한 비동기 Non-Blocking 처리
- **가상계좌 관리 서비스:** 은행사 TCP 전문 연동 프로세스 구축
- **외부 제휴사 연동:** 실시간 거래 내역 연동 API 및 Batch 프로그램 개발

각 도메인 간 느슨한 결합을 유지하면서도 비즈니스 요구사항을 충족시키기 위해 메시지 큐(RabbitMQ), 캐시(Redis) 등을 활용하여 안정적인 시스템을 구축했습니다.

### Q9. 보안을 고려한 개발 경험이 있나요?

**답변:**
금융 및 결제 플랫폼 분야에서 10년간 일하면서 보안을 항상 최우선으로 고려해왔습니다.

**주요 보안 강화 사례:**
- **회원 인증 프로세스 보안 강화:** NONCE 발급 및 검증 체계 수립으로 본인 인증 검증 데이터 재사용 방지, 로그인 비밀번호 암호화 데이터 재사용 방지
- **개인정보 보호:** 민생회복 소비쿠폰 시스템에서 개인정보 암호화 처리
- **ISMS-P 준수:** 탈퇴 회원 정보 분리 보관 및 보존 기간 만료 회원 정보 파기 프로세스 구축
- **KYC 체계 수립:** 회원 식별을 위한 KYC 체계 수립 및 AML 미수행/위반 회원 서비스 이용 차단
- **API 보안:** OpenAPI Gateway에서 Access Key 인증, HTTP Body 암복호화, Call-Count 제한 처리

코드 작성 시에도 OWASP Top 10 취약점(SQL Injection, XSS, Command Injection 등)을 항상 고려하며, 발견 시 즉시 수정합니다.

### Q12. 프로젝트에서 가장 어려웠던 기술적 챌린지는 무엇이었나요?

**답변:**
가상 순번 대기 시스템 개발이 가장 도전적인 프로젝트였습니다.

**챌린지:**
- 동시접속 100K 이상의 대규모 트래픽을 안정적으로 제어
- 실시간 대기열 순번 관리 및 공정한 진입 처리
- 좀비 토큰 자동 만료 등 운영 자동화
- p95 응답시간 3ms 이하의 초저지연 요구사항

**해결 과정:**
1. **아키텍처 설계:** Redis Sorted-Set 기반 대기열 관리 구조 설계
2. **성능 최적화:** Lua 스크립트로 단일 키 트랜잭션 처리, 락 최소화
3. **트래픽 제어:** Sliding-Window 알고리즘으로 진입 제어
4. **사용자 경험 개선:** HTTP Long-Term Polling + 서버 계산 기반 동적 Polling 주기 조절
5. **모니터링:** 실시간 트래픽 대기 현황 모니터링 및 통계 데이터 관리
6. **확장성:** AWS Auto-Scaling 환경 구성

**결과:**
- 분당 46K 요청 처리
- 단일 인스턴스 TPS 600
- p95 응답시간 3ms
- 월 1일 대규모 이벤트 기간 코어 시스템 지연 장애 0건

이 프로젝트를 통해 대용량 분산 시스템 설계, Redis 활용, 성능 최적화 등 다양한 기술적 역량을 크게 향상시킬 수 있었습니다.

### Q14. 레거시 코드 개선 경험이 있나요?

**답변:**
기존 플랫폼 서비스 운영 개발을 하면서 지속적으로 레거시 코드를 개선하고 있습니다.

**개선 방향:**
- **아키텍처 개선:** 헥사고날 아키텍처, 클린 아키텍처, DDD 등 설계 원칙 적용
- **리팩토링:** 코드 가독성, 유지보수성 향상
- **성능 개선:** 불필요한 DB Lock 제거, 캐싱 전략 적용, 비동기 처리 도입
- **보안 강화:** 인증 프로세스 개선, 개인정보 암호화

**사례:**
- 민생회복 소비쿠폰 시스템 개선 시 불필요한 DB Lock 조회 로직 제거
- 가맹점 연동 Batch 시스템을 동기 처리에서 병렬 비동기 처리로 전환하여 179배 성능 개선

단계적으로 개선하며, 테스트 코드를 작성하고 충분한 검증을 거친 후 배포합니다.

### Q15. 요구사항이 모호할 때 어떻게 대처하시나요?

**답변:**
요구사항이 모호할 때는 적극적인 소통을 통해 명확히 하려고 노력합니다.

**대응 방식:**
1. **질문하기:** 불확실한 부분을 구체적으로 질문
2. **시나리오 작성:** 예상되는 사용 사례를 정리하여 공유
3. **프로토타입:** 간단한 프로토타입을 만들어 피드백 수렴
4. **문서화:** 합의된 내용을 문서화하여 공유
5. **지속적 소통:** 개발 중에도 지속적으로 피드백 공유

**예시:**
- 가상계좌 연동 서비스 구축 시 프로젝트 리더로서 설계 단계에서 은행사 전문 규격, 거래 시나리오, 예외 처리 등을 상세히 정의하고 관련 부서와 협의하여 요구사항을 명확히 했습니다.

## 성장 마인드

### Q16. 개발자로서 가장 중요하게 생각하는 가치는 무엇인가요?

**답변:**
세 가지를 가장 중요하게 생각합니다.

**1. 지속적인 학습과 성장:**
- 반복적인 업무일지라도 조금이라도 다르게, 하나라도 새로운 것을 배워갈 수 있도록 변화를 주기 위해 노력
- 새로운 기술 트렌드에 빠르게 적응
- 스터디, 세미나 등을 통한 지식 공유와 함께 성장하는 문화

**2. 비즈니스와 기술의 균형:**
- 단순히 코딩만 잘하는 개발자가 아니라, 비즈니스 가치를 이해하고 기술로 구현하는 개발자
- 기술력으로 가치를 나타낼 수 있는 서비스 개발

**3. 책임감과 품질:**
- 유지보수 용이성과 확장성을 고려한 설계
- 테스트 자동화, 코드 리뷰 등 품질 관리
- 대규모 서비스의 안정성에 대한 책임감

**목표:**
사막을 묵묵히 한 걸음씩 건너가는 낙타처럼, 목표를 향해 정확하고 끝까지 나아갈 수 있는 개발자가 되는 것이 제 꿈입니다.

### Q17. 앞으로의 커리어 계획은 어떻게 되시나요?

**답변:**
**비즈니스와 기술을 연결하는 책임감 있는 개발자**, **기술력으로 가치를 나타낼 수 있는 서비스를 개발하는 개발자**로 성장하고 싶습니다.

**단기 목표 (1-2년):**
- MSA 환경에서의 분산 시스템 설계·운영 역량 강화
- 클라우드 네이티브 아키텍처 (Kubernetes, Service Mesh 등) 전문성 확보
- 대용량 데이터 처리 및 성능 최적화 경험 확대

**중장기 목표 (3-5년):**
- 테크 리드 또는 아키텍트로서 기술적 의사결정 주도
- 주니어 개발자 멘토링 및 개발 문화 개선 리드
- 오픈소스 기여 또는 기술 블로그를 통한 지식 공유 확대

**지속적 노력:**
- 새로운 기술 학습 및 실무 적용
- 팀과 함께 성장하는 문화 만들기
- 코드 품질과 서비스 안정성에 대한 끊임없는 개선

**5. 협업 및 커뮤니케이션:**
